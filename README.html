Інструкція з застосування програми для розсилки Реконограм

Як програма працює?

  1. Програма у постіному режимі здійснює моніторинг заданої у налаштуваннях директорії з заданим інтервалом часу.
  2. Якщо до директорії або її підпапок надходять нові файли, програма реагує на них:
    • Здійснює групування нових файлів за розширенням
    • Визначає отримувачів файлів за номерами Реконів
    • Формує тему та тіло листа
    • Робить розсилку сгрупованих файлів користувачам.
    • Формує відповідні повідомлення щодо статусу відправки
    • Переходить у режим очікування нових файлів.
  4. За виникнення помилок у роботі робить їх розшифровку.
  5. Доступ до файлів здійснюється локально без застосування протоколу FTP або бази данних.

Програма містить наступні виконавчі файли:

  1. ReconMail.exe - файл програми (запуск та зупинка, працює у фоновому режимі)
  2. Settings.xml - файл конфігурації (для задання налаштувань)
  3. errors.txt - лог-файл помилок, що виникають при роботі програми та розшифровуються у файлі
  4. sent_messages.txt - лог-файл у який записуються повідомлення про успішну відправку пошти
  5. Всі інші файли призначені для роботи програми, їх не чіпати.

Перед початком роботи:

  1. Внести налаштування у Settings.xml або перевірити, що все налаштовано правильно.
  2. Видалити усі повідомлення щодо помилок у файлі errors.txt
  3. Запустити виконавчий файл ReconMail.exe, програма починає моніторити задану директорію
  4. За необхідності зупинки програми дати команду CTRL + C

Як налаштувати програму?

  1. Відкрити файл settings.xml будь який текстовим редактором або редактором коду.
  2. Внести налаштування між тегами:
    • server - адреса SMTP серверу (НЕК - mail.ua.energy)
    • port - порт, що використовується сервером (НЕК - 25)
    • use_auth - потрібна аутентифікація чи ні (НЕК - False)
    • username, password - задати якщо потрібна аутентифікація
    • use_SSL - застосовувати захищену передачу пошти (протокол SSL), якщо False то буде використовуватись протокол TSL
    • sender_name - імя створеного серверу розсилки (будь-яке)
    • sender_email - пошта створеного серверу розсилки (будь-яка)
    • signature - підпис листа (дод інструкції, або типу "З повагою, сервер Рекон...")
    • message - текст повідомлення, що буде відправлено
    • send_messages - відправляти повідомлення чи ні (якщо True - то буде)
    • recipient - дані отримувача:
      • email - пошта
      • files - номери Реконів які він буде отримувати ("1,2,3,4 ...") або "*" - якщо всі
    • directory - абсолютний шлях до загальної папки з реконами, що моніториться
    • interval - інтервал часу через який програма запускає моніторинг
    • from_date - з якої дати почати моніторити файли, у форматі "dd.MM.YYYY" (за замовчуванням з поточної дати)
  3. Зберегти зміни, закрити файл, запустити програму.
